# QA Automation Framework

A scalable, production-ready automation framework built using **Python** and **Pytest** for 
testing **Web, Android, iOS, and REST APIs** from a single codebase.

This project demonstrates industry-standard automation practices used in modern QA teams, including:

* Cross-platform automation (Web, Android & iOS)
* Page Object Model (POM)
* Pytest fixtures and configuration management
* Parallel execution with pytest-xdist
* Data-driven testing
* API testing
* Logging and reporting
* CI/CD ready architecture
* Reusable utilities and helper modules
* Easy configuration for multiple environments and devices

## Tech Stack

* Python
* Pytest
* Selenium WebDriver
* Appium
* Requests
* Pytest-xdist
* Allure Reports
* Git & GitHub
* Jenkins (CI/CD)

## Project Goals

The objective of this project is to create a maintainable, scalable, and reusable automation framework 
that follows QA engineering best practices and can be extended for enterprise applications.

The framework is designed to support:

* Independent execution of Web, Android, and iOS tests
* Parallel execution across multiple browsers and devices
* Easy addition of new platforms and test suites
* Clean separation of test logic, page objects, utilities, and configurations
* Cross-platform code reuse wherever possible

This repository serves both as a learning project and as a portfolio demonstrating automation framework 
design and implementation.
